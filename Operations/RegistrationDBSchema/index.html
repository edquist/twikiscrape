Proposed Resource Schema

Relations: (Resource) Support Center id<-> (Support Center) Support Center id Relations: (Resource) Resource id <-> (Support Center) Support Center Short Name

resource table

Changes made by Mine Altunay in green -- attribute definitions

  • resource_id : id generated by the GOC database
  • site_id : this will come from the pull-down Site menu. Resource will select the Site upon registering
  • url : This is a website that describes the resource
  • name : resource name
  • OSG Grid : type of grid in which the resource resides, production or integration
    • OSG Production
    • OSG ITB
  • type : resource type, either compute or storage
    • Compute Element
    • Storage Element - for storage elements, we'll need to add ldap host/port and search string
  • Support Center id number : support center is selected from a pull down menu, specified at registration
  • Date Submitted (Auto Populated)
  • Submitter Name (Auto Populated from Cert)
  • Submitter Email : the person who registers the resource

gatekeeper table

  • resource_id :resource id same as above
  • Resource FQHN :resource IP address

resourcecontact table

  • person_id :person id generated by the GOC db
  • resource_id :resource id same as above
  • type :4 types: manager, operational, security contact, or incident response contact. each contact type can have alternate (two people listed per type, see rank)
  • rank :shows if this person an alternate contact or not.
  • details :text, what are possible details ???
  • cert :X.509 cert of the contact

vo_resource_ownership

  • resource_id :resource id being owned same as above
  • vo_id :VO owning the resource, same as above
  • percent :the percent of the resource owned by the VO

Proposed Virtual Organization Schema

Pre Approval Submission

  • vo_id :VO id generated by GOC db
  • VO Short Name :VO name
  • Community :description of user community
  • Application Description :type of application being run by this VO
  • Date Submitted (Auto Populated)
  • DN of Submitter (Auto Populated by Cert)
  • Submitter Name (Auto Populated by Cert)
  • Submitter Email :the person who registers the VO

Post Approval Submission Relations: (Virtual Organization) Support Center Name<-> (Support Center) Support Center Short Name

virtualorganization table

  • VO Primary URL :VO web page
  • VO Privilege URL :the list of VO internal groups and the corresponding access rights for each group. This is posted on the privilege URL , see twiki/VO pages for example
  • Membership Services URL (VOMS URL) :VOMS address
  • VO AUP URL :the VO Agreed Usage Policy, posted on this URL. This will not be submitted right away, but later when the VO creates its AUP, this will get filled out
  • VO Purpose URL :statement of VO purpose. This is a subset of VO AUP. Since AUP is more comprehensive (with rules and restrictions over the user), this statement only covers what is the common goal of this VO members
  • User Support URL :VO maintained support web page for its users -- this is not a Support Center
  • Support Center id :VO's support center. This is different from support URL. the VO selects the Support center from a pull down menu.
  • Date Submitted (Auto Populated)
  • DN of Submitter (Auto Populated by Cert)
  • Submitter Name (Auto Populated by Cert)
  • Submitter Email

Services

Important note form schema freeze meeting on 10/17/2007: Service is expected to be owned by a VO or a resource, or both. We should provide additional interface at GOC registration web site so that while registering a VO or a resource, the services must also be listed at the GOC registration web page.

service table

  • service_id
  • site_id a service can be owned by a site or a vo. This is the owner site's id. Note that a service can have simulatenously be owned by VO or site.
  • vo_id a service can be owned by a site or a vo. This is the owner VO id. Note that a service can have simulatenously be owned by VO or site.

servicecontact table

all attribute definitions are same as resource contact table, except site_id and vo_id are additional and explained below

  • person_id
  • service_id
  • site_id owner site id
  • vo_id Note this attribute is added on 10/17/2007, owner vo_id

  • type
  • rank
