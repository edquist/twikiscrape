Hot Topics for the week

Large OSG resource request (Steven Clark)

nanoHUB has an application that they'd like to run on the grid. Jobs man run between 6-10 days, or an average of 8 days; 1800 jobs to run to sites that have Condor. Alain Roy with the VDT wonders if anyone has a policy that would be a problem for jobs that are going to run this long. CMS will bump up the allowed simultaneous nanoHUB jobs on their cluster to 25. The jobs will come in groups of 10. Not all of these jobs would be submitted to one site, but they'd prefer to do so. 6-9 sites support nanoHUB now.

Leigh asked what is the longest running CMS job? Someone mentioned that it was perhaps 2 days. So can jobs even run for as long as 8 days? Steve will go through the list of sites that support nanoHUB and review their policies, then submit jobs. The plan is be ready to start running jobs by tomorrow. Neither UFlorida-IHEPA or VAMPIRE-Vanderbilt sites accept nanoHUB jobs; authentication fails. The GOC is tracking this in a ticket and we'll watch for more followups.
