Rules for topics and template

  • All topics have a bold heading at Level 1 (---+*Title*) and first letters of words are capitalized (eg. Rules For Formatting Documents).
  • The level one heading is taken as its twiki topic name, separated into words (e.g., MyTopic becomes My Topic). Choose the twiki name carefully!
  • All level two sections should be in title case. Eg: ---++ Details on the Installation (except for proper nouns)
  • All sections (level three and below) have the first word capitalized, the rest lower case. Eg: ---+++ Details on the installation (except for proper nouns)
  • Exclude from the TOC all sections level three and below (---+++, ---++++, etc.). To accomplish this, use %TOC{depth="2"}%
  • Include COMPLETE, RESPONSIBLE, and REVIEW directives at the bottom.
  • Trash.ReleaseDocumentationExampleMainTopic for a main topic not likely to be used with STARTINCLUDE/STOPINCLUDE so as to be included in other topics.
  • ExampleTopicInclude for a topic likely to included in other topics, so STARTINCLUDE/STOPINCLUDE and EDITTHIS directives are used.
  • Trash.ReleaseDocumentationExampleCompositeDocument what a composite document of includes should look like.
  • Wherever possible TWiki editing should be used rather than raw HTML.
  • Avoid nested INCLUDES.
  • Here is the WebTopicEditTemplate for newly created pages in this twiki, which are assumed to be included in other topics.
  • For content that is OSG 1.2 specific, use the STARTOSG12 variable to before the content and ENDOSG12 after the content. E.g. %STARTOSG12% OSG 1.2 content %ENDOSG12%. You need to place a space between the %STARTOSG12% tag and the content.

Screen Views

Many documents refer to unix commands, output from a script, a set of lines that should be edited in a file (like a configuration file), or a program listing. These should be set off in a standard way as follows:

For a set of commands that can be copied and input or a dump of the screen results from running a script uses a "screen" view: 

<pre class="screen">
command 1
command 2
</pre>
which looks like: 
command 1
command 2

File View

For a set of lines in a file to be edited or the contents of a file uses a "file" view: 
<pre class="file">
line 1
line 2
</pre>
which looks like: 
line 1
line 2

Variables

See WebPreferences for settings of the following variables.

  • OSG release variable: %OSG_VERSION% = 1.2.28
  • ITB release variable: %ITB_VERSION% = 1.1.32
  • VDT release variable: %VDT_VERSION% = 2.0.0
  • Release variable, according to cache (ITB or OSG): %VERSION% = 1.2.32
  • VDT cache variable: %VDT_CACHE% = vdt_200_cache
  • ITB/OSG cache/grid variable: %CACHE% = http://software.grid.iu.edu/osg-1.2
    • Use the %CACHE% variable as a generic reference for either "ITB" or "OSG" caches, grids, etc. It automatically expands to "ITB" if its in the Trash/Trash/Integration web, or to "OSG" if its in the production release documentation web.
  • Refer to the cache/version combination that you're using, e.g, http://software.grid.iu.edu/osg-1.2 1.2.32, with a space in between, as
    %CACHE% %VERSION%
  • Full URL to base of VDT Documentation for current release %VDT_DOCS_URL%: http://vdt.cs.wisc.edu/releases/2.0.0
  • Full URL to VDT Cache for this release: %VDT_CACHE_URL%: http://vdt.cs.wisc.edu/vdt_200_cache
  • Full URL to OSG Cache (ITB or Production) for this release %OSG_CACHE_URL%: http://software.grid.iu.edu/osg-1.2/
  • Pacman version: %PACMAN_VERSION%: 3.28
