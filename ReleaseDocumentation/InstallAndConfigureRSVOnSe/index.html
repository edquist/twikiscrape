Install and Configure RSV on OSG SEs


RSV background

The OSG-RSV package does not come by default with any of the OSG Storage element packages (it is included by default with the OSG CE). You will have to install it separately, preferably in a different directory from the SE itself. Separating the installation from your SE tools makes it easier to update the components separately.

RSV Installation

Instructions to install RSV. Modify the installation path below as desired: 

 mkdir /opt/rsv
 cd /opt/rsv
 pacman -get http://vdt.cs.wisc.edu/vdt_200_cache:OSG-RSV
 . setup.sh

Once downloaded, additional configuration steps are necessary as documented below.

RSV running on a storage element door that runs on an OSG-CE node

If your storage element door runs on an OSG-CE node, then you can configure RSV using the configure-osg script, and its associated config.ini file.

Edit the [RSV] section in the config.ini -- each variable in the section is commented upon in detail. Or you can also refer to the RSV section of the config.ini twiki page.

Using an existing Apache server on machine to host RSV info

If you already have an Apache server running on the machine you install RSV on, then you can configure it to serve RSV local result web pages.

Disable the Apache that was installed with RSV. Assuming RSV was installed at /opt/rsv: 

cd /opt/rsv
. setup.sh
vdt-control --off apache
vdt-control --disable apache

Add the following stanza to your httpd.conf file. Make sure to correct the bolded path below to point at your RSV installation: 

# Set options for RSV's HTML directory
<Directory "/opt/rsv/osg-rsv/output/html">
    Options None
    AllowOverride None
    Order Allow,Deny
    Allow from all
</Directory>

# Create alias for RSV output directory
Alias /rsv /opt/rsv/osg-rsv/output/html

Then force Apache to reload its configuration, or restart it.
