Security (Mine)

  • Vulnerabilities/Incidents
    • Incident report from FNAL-FermiGrid. Thought as DOS attack, but turned out to be student job consuming memory. Two underlying problems caused the issue. A VO should not keep sending jobs on behalf of a user who fails authN at a site. Pilot jbs somehow increase their memory footprint even after a user fails glexec.
    • IRC botnet announcement. Several IRC bots were placed on US universities that are also members of OSG. Grids-sec contacted the universities. No compromises on osg grid sites. No further feedback form sites.
  • DigiCert? testing in ITB. Expected to finish by next week. No component failure so far (CS/GUMS /VOMS tested).
  • DOEgrids root CA expiring. New root CA has been released to IGTF. OSG delayed adopting the new release without testing the cert in ITB. Doing a test in ITB with a few sites which are not doing DigiCert?. Two sites completed tests. Waiting for Suchandra to do tests. Expected production release is Dec 14th. ESnet today reported they plan to start using the new root cert in production on Dec 14th. This may cause problem with our release date. We are responding asap.
    • Examining problems from end user perspective. We found notes from last time DOEGrids updated root certs. It caused problems in browsers. We are examining whether the same problems still exists and how we can mitigate them before reaching the end user.
