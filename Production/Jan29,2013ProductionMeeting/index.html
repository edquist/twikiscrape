  • Operations
    • Pakiti made it into software release. Next steps are to raise site admins awareness of this tool. We plan for training sessions at AHM.
    • Traceability requirements document is near completion. The new requirements allow sites to run grid jobs without any end user certificate as long as the VO meet the traceability requirements. Fermilab security officer is reviewing the requirements and already gave a feedback over the first draft. V2 is completed and sent to Fermilab security team. Once Fermilab accepts these requirements this will open the door for VOs to run jobs without certificates over Fermilab grid resources.
    • New problems concerning Digicert certs are found. Dave Dykstra and others are working on the issues. If a worker node uses a host cert proxy (not the real host cert), connection to GUMS fails. This also impacts the case where worker node has no certificates at all and uses the pilot proxy to connect to the GUMS. This is an important issue becuase cms sites were getting ready to upgrade to latest gums/glexec and stop having certificates on worker nodes.
