  • Problem with BeStMan clients, GridFTP server, and OpenJDK 1.7.0_45
    • Client connects to server, but then hangs up immediately 
      The GSI XIO driver failed to establish a secure connection. The failure occured during a handshake read.
System error in recv: Connection reset by peer
A system call failed: Connection reset by peer
    • Root cause: Change in Java security setting rejects RSA keys < 1024 bits; grid-proxy-init defaults to 512
    • Unofficial workaround #1: Edit /usr/lib/jvm/java-1.7.0-openjdk-1.7.0.45.x86_64/jre/lib/security/java.security: 
      jdk.certpath.disabledAlgorithms=MD2, RSA keySize < 1024
    • Unofficial workaround #2: Use grid-proxy-init -bits 1024 (or higher, see -help)
