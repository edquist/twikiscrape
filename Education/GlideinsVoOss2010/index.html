glideinWMS for the VO administrator - Hands-on Session, Wednesday July 21st, 2010

Work in progress

This session will give you hands-on experience in using the glideinWMS as a VO frontend administrator.

Session format

Installing a VO frontend is a lengthy process, so it will not be attempted in this session.

Instead, the students will look at an existing frontend installation and will be guided through the various configuration options by the instructor.

glideinWMS Documentation

  • glideinWMS Home Page
  • glideinWMS Manual
  • glideinWMS VO Frontend Manual

Local Condor setup

As with all the other hands-on exercises, we will be using
vdt-itb.cs.wisc.edu
to submit and monitor our jobs.

We will be again using the glidein Condor setup, so if you have not done it yet, please run: 

[sfiligoi@vdt-itb ~]$ source /opt/glidecondor/condor.sh

At any given point, you can check you are using the right one by using: 

[sfiligoi@vdt-itb ~]$ which condor_submit
/opt/glidecondor/bin/condor_submit
[sfiligoi@vdt-itb ~]$ echo $CONDOR_CONFIG
/opt/glidecondor/etc/condor_config

Local Frontend setup

The local frontend is installed in 

/home/frontend/frontstage/frontend_OSGSchool-v1_0

It can be monitored at http://vdt-itb.cs.wisc.edu/vofrontend/monitor/frontend_OSGSchool-v1_0/frontendStatus.html.

Frontend configuration

The Frontend configuration is almost entirely contained within frontend.xml.

Have a look at it (compare to the manual)
