glideinWMS for the factory administrator - Hands-on Session, Wednesday July 21st, 2010

Work in progress

This session will give you hands-on experience in using the glideinWMS as a factory administrator.

Session format

Installing a glidein factory is a lengthy process, so it will not be attempted in this session.

Instead, the students will look at an existing factory installation and will be guided through the various configuration options by the instructor.

The interaction between the factory and the frontend will also be analyzed.

glideinWMS Documentation

  • glideinWMS Manual

Local Condor setup

As with all the other hands-on exercises, we will be using
vdt-itb.cs.wisc.edu
to submit and monitor our jobs.

The glidein factory using yet another Condor installation. In order to see it, please run (in a clean terminal session): 

[sfiligoi@vdt-itb ~]$ source  /home/gfactory/glidecondor/condor.sh

At any given point, you can check you are using the right one by using: 

[sfiligoi@vdt-itb ~]$ which condor_q
/home/gfactory/glidecondor/bin/condor_q
[sfiligoi@vdt-itb ~]$ echo $CONDOR_CONFIG
/home/gfactory/glidecondor/etc/condor_config
[sfiligoi@vdt-itb ~]$

Local Glidein Factory setup

The local frontend is installed in 

/home/gfactory/glideinsubmit/glidein_v1_0

It can be monitored at
http://vdt-itb.cs.wisc.edu/glidefactory/monitor/glidein_v1_0/factoryStatus.html and
http://vdt-itb.cs.wisc.edu/glidefactory/monitor/glidein_v1_0/factoryStatusNow.html.
