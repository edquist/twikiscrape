2. Analysis of the results: 

a. lcg_util now works better than before, but reveals there is a difference in the semantics of GlueSAPath - 
in EGEE we would have GlueSAPath /pnfs/fnal.gov/data/opssgm, while in OSG they have 
only /pnfs/fnal.gov/data and expect the user to guess that he must write only under opssgm. 
Seems to be the same problem for all OSG sites.

In some cases the directory is writable anyways, and lcg_util works, but in other cases it attempts to write 
in places it is not allowed
This is an important point to be discussed.
  • Emanouil has sent an e-mail describing the details of the issue
  • Anthony will go through it --> task 23 opened 
b. the tests reveal that the gLite software is not available (not in the PATH) in the worker nodes - 
see how commands like lcg-cr or glite-brokerinfo are not found.
To be investigated by Andreas Unterkircher.

c. One SRM SE now works with SRM copy - fapl033.fnal.gov.
BNL SRM SE does not respond on port 8443.
Authentication problem at UCTier3 - solved yesterday, but infosystem does not display properly ops support - 
new iteration started. This is not an SRM SE.
  • being followed up
