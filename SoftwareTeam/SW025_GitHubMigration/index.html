Project 25: Migrate SVN repositories of OSG-owned software to Git and GitHub

Description

Background

GitHub provides a useful collaborative environment for software projects. Many OSG-owned software projects are currently only available in SVN. We can migrate the repositories for those projects to GitHub and gain access to its collaborative features.

Goal and Scope

All projects that are in the VDT SVN software area and are still active will be migrated to GitHub: a new GitHub repository will be created per project, and the original SVN directories will be closed to further changes.

Links

  • Migration proposal

Tasks

Migration occurs in three phases:

  1. Schedule a time with other developers to ensure minimal interruption to their workflow.
  2. Migrate their repos.
  3. Verify functionality and work with developers to resolve issues.

We don't want to move projects by "forking" them on GitHub because that results in screwy metadata. Instead, use the following move procedure:

  1. If the repo already exists under opensciencegrid, rename it with a .orig extension.
  2. Create a new repo under opensciencegrid
  3. Mirror-Clone from the old repo
  4. Mirror-Push to the URL of the new repo

For example, with OSGConnect/StashCache:

  1. Create opensciencegrid/stashcache in the web interface 
  2. git clone --mirror https://github.com/OSGConnect/StashCache.git
  3. cd StashCache.git
  4. git push --mirror git@github.com:opensciencegrid/stashcache.git

git push will complain about pull request refs getting rejected (by github), eg: 

! [remote rejected] refs/pull/1/head -> refs/pull/1/head (deny updating a hidden ref)

Alas, we lose the pull request history (as well as github issues) from the original project.
