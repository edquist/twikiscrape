Chris: fixed a problem with the old-style ps-accounting reporting reported by Keith which turned out to be a typo in a .rptdesign file.

Chris: also fixed a problem with summary table generation for ps-accounting. Believed to be due to an inefficient cursor creating a temporary table which blew /tmp. Replaced with a cursor which does the necessary grouping, requiring only one insert per summary table row. One possible enhancement might be to do the whole table insert in one suitably-grouped select. It does bring up however, the whole idea of when summary tables should be generated and/or updated, and what to do about information that may evolve, like CPUInfo values for a farm -- regenerating old table information would at the moment use newer CPUInfo data for each node and give a possibly misleading history. May want to consider putting validity information into CPUInfor to address this problem.

Probes

Chris: fixed a problem with large backlogs being slow to recover due to inefficient backup filename choice mechanism. Changed to mktemp(1) (not mktemp(3) which suffers from a race condition) instead of cycling through the numbers (in extreme cases can lead to >5000 stat operations before a new record is processed).
