Probes

John: Assisted Chris in testing viability of capturing DN/FQAN data in the JobManager? rather than using GRAM Auditing.

  • GRAM2 (pre-ws) always has the proxy available.
  • GRAM4 (ws) only has the certificate proxy when a delegated proxy is used. If you use globus command line submittals (globusrun-ws), you have to specifically request delegation.
  • CondorG? submittal always use delegation. However, we were 50-50 on getting these to work. They would work for me, but not Chris. We still don't know why.
  • Another factor in use of CondorG? is an 80% increase in the number of authorization requests to the GUMS authz service. For a normal globusrun-ws, there are 5 authorizations. For CondorG?, there are 9.

Steve: There are some tricky client configurations--I believe the underlying cause of Chris' problems on fnpcsrv1 is that /etc/grid-security/gsi-authz.conf specifies a prima callout and the java gridftp client used by condor-G looks at that. Chris has asked us to look at this.
