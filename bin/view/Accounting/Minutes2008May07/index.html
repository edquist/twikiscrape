John W:.Found the reason why we keep getting a new user defined in the mysql.user table as gratia@% with no password. This causes one to fail on login attempts as user gratia as it collates up first when MySql? is verifying logins.

It is the "GRANT SELECT ON mysql.proc TO gratia;" in the create_build-stored-procedures-sql script. When doing this MySql? automatically creates the user if it doesn't exist.

Since this is needed only for local development we need to find an alternative way of doing this. Have not come close to resolving this.
