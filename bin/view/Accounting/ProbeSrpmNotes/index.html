OSG Accounting Services - Gratia - Rebuilding from source RPM.

If you wish to re-build from the source RPM (say for a particular architecture or software environment or with a different default base prefix to obviate the need for the --relocate option), then:

  1. Optionally set a non-root build area for your RPMS by building a replica tree of /usr/src/redhat in your own area and creating a file ~/.rpmmacros that contains the line: 
    %_topdir <my-rpmbuild-dir>

  2. The following macro definitions may be used on the rpmbuild command line:
    --define='config_itb 1'
    ProbeConfig files for new installs will have defaults suitable for ITB nodes.
    --define='vdt_loc /usr/local/vdt'
    Specify a different location for the VDT install.
    --define='default_prefix /usr/local/vdt/gratia'
    Specify a different location for the gratia installation (not necessary if default_prefix = %{vdt_loc}/gratia).
    --define='site_name <pretty-site-name>'
    Specify a particular site name rather than picking from VDT configuration or leaving generic.
    --define='meter_name <meter-name>'
    Specify a particular probe name rather than building from uname -n (most appropriate for the ps-accounting probe which needs to be installed on all worker nodes).
    --define='python python'
    Pick up python from the path and remove the requirement on the version number of the system's python RPM; OR
    --define='python /my/path/to/python'
    Hard-code a python path into the RPM.

  3. Two build commands are required: one for the non-architecture-specific RPMS, and one for the local architecture. A suitable composite command could be: 
    for arch_arg in "" "--target=noarch"; do
  rpmbuild <defines>; $arch_arg --rebuild <src-rpm>
done
    Note that upgrades leave existing configuration files untouched.
