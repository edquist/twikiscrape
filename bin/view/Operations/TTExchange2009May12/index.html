These tickets come into our system and creates some ticket Fermi/Remedy - INC00000000#### - ISSUE=FOO PROJ=71 which is to say that the subject we get from remedy looks more like this "Fermi/Remedy - INC000000001526 -" and that footprints ticket FOO is totally empty, with exception of the message: A New Fermi/Remedy ticket has been created. Fermilab/Remedy HelpDesk? ticket was created at m/dd/yyyy hh:mm:ss AM/PM for User bridge, cd-srv.

This message does not include any references to a footprints ticket with content, nor any decipherable problem report or instruction. These tickets are neither populating an originating ticket ID nor a destination ticket ID. If the idea is that we now, after the upgrade, must populate these fields by hand, that is something that was never clearly communicated to us, and something we had never done before the upgrade.

Note from Rob: Populating fields by hand is not an option. The GOC does not have effort for this, which is why the ticket exchange was created.
