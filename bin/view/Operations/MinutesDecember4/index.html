Operations Agenda

Followup items from the previous meeting

1) Deprecating grid-proxy, use voms-proxy

  • Started with the desire to execute apps on a server as the user of the app instead of root or apache. Fermilab and a group from the Netherlands worked on...have to use voms-proxy...no way to tell what VO a person should be in with grid-proxy w/o GUMS. So the suggestion is to move away from grid-proxy to voms-proxy. Dan sent email earlier today to osg-general and osg-et detailing the reasons why we should consider this change. See the email for more info. Eric has invited Dan to attend the Troubleshooting Meeting today following the Operations Meeting to allow for further discussion.
  • SAZ only accepts voms-proxies...it allows grid-proxies in the db, but it denies them automatically and to be added you have to be manually added by the SAZ admin. SAZ is not in the VDT, there has been no request to have it added.
  • If we use voms-proxy instead of grid-proxy, what would we break? Right now we're not sure.

2) Large resource request for nanoHUB update (ongoing) - http://osg.ivdgl.org/twiki/bin/view/VO/NANOHUB_UtilStatus

  • Last update: nanoHUB is now running on Buffalo sites. Not running jobs on Vanderbilt, but they are red on Gridcat. GOC ran the failing Hello World test manually and it passed. Leigh had some questions about how to understand the maps on the nanoHUB stats page.

  • This week: Having authorization issues at UNM and Vanderbilt. Vanderbilt has problems with their new gatekeeper and since then nanoHUB has had problems there. I spoke with Jim Thomas at UNM today. Other sites are also having problems with authorization failures and they're going to look into those.
