Post Upgrade Functionality Checks

This document is to provide a list of checks that will test the functionality of the MyOSG service after an upgrade has been done. It is assumed the DNS round robin (RR) mechanism was used during the maintenance period and these checks should be performed before an instance is returned to round robin.

  • Verify which instance has been removed from RR with dig.
    • myosg1.grid.iu.edu is 129.79.14.169
    • myosg2.grid.iu.edu is 134.68.22.137
    • dig, for example, returns this: 
2011-06-10 14:31:59 UTC [steige@jump:~]$ dig myosg.grid.iu.edu

; <<>> DiG 9.3.6-P1-RedHat-9.3.6-16.P1.el5 <<>> myosg.grid.iu.edu
;; global options:  printcmd
;; Got answer:
;; ->>HEADER<<- opcode: QUERY, status: NOERROR, id: 1050
;; flags: qr rd ra; QUERY: 1, ANSWER: 2, AUTHORITY: 0, ADDITIONAL: 0

;; QUESTION SECTION:
;myosg.grid.iu.edu.             IN      A

;; ANSWER SECTION:
myosg.grid.iu.edu.      2986    IN      A       129.79.14.169

;; Query time: 11 msec
;; SERVER: 192.168.96.4#53(192.168.96.4)
;; WHEN: Fri Jun 10 14:34:06 2011
;; MSG SIZE  rcvd: 67

  • Note that 134.68.22.137 is not in the "ANSWER SECTION" returned from dig. This means myosg2.grid.iu.edu is the instance removed for maintenance and is the instance to be checked.

Functionality

  • Login to myosg1.grid.iu.edu or myosg2.grid.iu.edu and become root.
    • Execute /opt/service-monitor/myosg/test.sh If problems are found e-mail will be sent.
  • go to the correct instance: MyOSG1 or MyOSG2
    • page should load in a resonable time, without error and with the expected content.
  • click on the "Sample Pages"
    • page should load in a resonable time, without error and with the expected content. The "Status Map" page is an exception, it will only load correctly from MyOSG (without the 1 or 2)
    • "Current RSV Status" page should contain recent data
  • do an OIM search. Enter text (csiu, for example)
    • search results should appear below the search box
  • Make sure there are no service monitor alerts posted for goc-alert for at least 30 minutes after the maintenance completion.
