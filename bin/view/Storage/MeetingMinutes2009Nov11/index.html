Ted: 

For discovery tool:

made fixes to make attribute comparisons non-case sensitive and to allow functions in xquery variable declarations.
Modified the maven build to produce a product for either the release branch or the trunk. The former has 
its version identified with format x.x.x and the latter has its version identified with format x.x-rxxx, 
where xxx is the svn revision number. 
Also added a version variable in the readme text which is replaced by the true version number at build time. 
A new change log has been added with the same feature. The xpathsearch jar, based on the java code of the tool,
has been given it's own version number, since a new discovery tool version might be based on changes 
to non-java files, such as xqueries. Will likely add a version variable to the query scripts, 
so that a --version flag could be honored.

Looked at Complex Event Processing as a technology for a diagnostics tool.

Neha:

permissionPolicyEnforcementPoint=PnfsManager so, i added support for it in our conf file and script

validated the fresh 1.9.5-5 install on fg0x1, investigated problems bringing up srm
