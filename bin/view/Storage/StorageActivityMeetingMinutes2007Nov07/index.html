Storage Storage Activity Meeting Minutes 2007 Nov 07

Attending

Ted Hesselroth, Neha Sharma, Tanya Levshina, Alex Sim.

Discussion

Open Tickets - Neha
  • 4100/4115 gsidcap authorization bug. dCache ticket #2139. Forwarded to dCache developers, no dCache ticket # yet.
  • 4153 - Pool throws NullPointerException? when p2p copy requested by replica manager. dCache ticket #2107. Fixed in 1.8.0. Tanya will confirm that it is fixed in 1.7.0-47 before closing the ticket.
  • 4207 - dCache client unable to retrieve files (from certain pools NOT all) which have replicas. dCache ticket #2144. Waiting on update from DESY.
  • 4227 - srm.batch ignoring gPlazma parameters. Fixed in 1.8.0. Ted will confirm that it is fixed in 1.7.0-47 before closing the ticket.
  • 4228 - bidirectional gridftp adapter not working. No update.
  • 4246 - p2p transfers failing for several files in dcache-server-1.7.0-36. Site is monitoring. Possibly fixed in 1.7.0-37.
  • 4271 - dCap transfers being suspended (~10/hr) with "Pool Unavailable" error. User will upgrade after SC'2007 and see if problem exists.

Installation - Ted
A preview version of the VDT packaging of dCache 1.8 has been created for sites to experiment with. The scripts only support a fresh install. Further work will be needed to support upgrades from 1.7 to 1.8. Tanya said that some parts of StoragePoolManager?.conf have been deprecated. Ted will make the preview package available on the VDT web site. A new version of dCache, 1.7.0-47 has been release. Neha will add a feature to install_pnfs.sh to search for rpcinfo if it is not found in the path. That change and the new dCache patch will be put in the next VDT packaging of dCache production version. Neha will test that package for upgrades on the ITB fapl test stand. Discussion of supporting lcg-utils. See Testing.

Extensions - Ted
Discussed opportunistic storage via space reservations in dCache by associated a rootpath in the namespace with a set of pools, and having link for them authorization for the VOs. Users will be able to make space reservations within those pools. The reservations will expire and the files cleaned up using Greg Sharp's space reservation cleaner tool, thus recycling the space. Ted will write instructions, and consider methods of preventing abuse of the storage, such as only making reservations possible using an intermediate tool which limits reservation size. Discussed how opportunistic storage would work with Bestman. Alex described how it could be done in User-controlled space using unix file permissions and quotas.

Testing - Tanya
Neha will modify load test scripts to use lcg-utils to check performance on GridWorks? test stand. Ted will supply the rpms and instructions for installation. Tanya will begin working on scripts to test space reservation.

Bestman - Alex
Working on srm demo for Supercomputing 2007.

Tools - Abhishek
No report.
