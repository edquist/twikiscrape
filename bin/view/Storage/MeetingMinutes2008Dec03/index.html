From Ian:

"We continue to have problems with our SRM setup. On -Exp we get the 300 second timeout, but the file is being transferred and on the actual system you can see the file. Similar problem with GridFTP? and actually files are zero length files, so perhaps there is something wrong if data the files are supposed to have some content -- in fact, I would expect them to contain something so as to test the transfer of data between sites."

CDF:
++++++

Opportunistic storage being evaluated successfully at UCSD site. VO Group in OSG to start provisioning procedure with CMS and ATLAS to get more sites for CDF.

From Manoj:

"I tried to use the Nebraska SE. I was not able to transfer file using the Fermi SRM client (srmcp) between SRM at UCSD and Nebraska. Hence, it is not feasible to use the Nebraska SE in our case. Since the SAM-SRM interface is based on Fermi SRM client. This is also some thing the OSG storage people would have to look into it. An application being developed using a particular SRM client should be portable on all the flavors of SRM. Otherwise, the users would have to restrict themselves for the particular flavor of SRM on which its SRM client is supported."

comment from Brian:
The SAM-SRM interface does support what Manoj is trying to do; Andrew B has indicated that it's either in a later version of the SAM-SRM interface or requires a more up-to-date Fermi SRM client. I forget which.

comment from Steve:
Manoj has had a horrible time in general trying to get SRM to work because he is trying to get srmcp shoehorned into a legacy CDF SAM setup which requires the use of ups/upd. He's not getting any help from srmcp developers or from the CDF software people and he has been at this for months. Someone needs to step up to the plate and help him out.

CMS: ++++++

From Malina:
"With this level of unreliability (and I'm told, unscalability), has any thought been given to making it possible to do third party transfers from dCache->BeStMan without pushmode? I haven't a clue what the impact is of what I'm asking, so if this is something that would be difficult (impossible?), certainly discard the suggestion!"

comment from Brian:
It is possible to do third party transfers from dCache to BestMan? without pushmode. This is how the Nebraska T2 operates, and it scales sufficiently for our needs (peaks up to 10Gbps, sustains a couple Gbps). It cannot be done with the Fermi SRM clients (this is somewhat related to Manoj's problem) because the Fermi SRM client does not support that part of the protocol. It requires either LBNL SRM client or an FTS server / FTS client.
