Seven agenda item

Tickets

Discussion

No tickets were entered to GOC. Andrew has been declined to get access to GOC (Tanya is trying to resolve this issue). Ted didn't have a chance to do it so far. The following problems/questions have been reported during last week:

  1. Problems with vdt-dcache-SL4_32-2.2.8 upgrade at MWT2 (reported by Charles Waldman)
    1. java will not start after upgrade - looks like some problems with symlinks - restart of dcache helped to solve the problem
    2. After the upgrade, we did '/etc/init.d/dcache-pool restart' on the pool nodes. This failed to stop the running pools, and started a second Java process. This resulted in corruption of the 'companion' database, which took some time to repair.
    3. Nodes which are running both doors and pools are incorrectly set to NODE_TYPE=door (in node_config) instead of NODE_TYPE=custom. This results in the pool services not being upgraded/installed.
    4. A '-dryrun' install of pnfs resulted in a script /opt/pnfs/bin/pnfs which calls itself. We reverted to the older version.
    5. The deprecated scripts 'dcache-core' and 'dcache-pool' are not removed from the runlevels.

  1. Transfer timeouts from MWT2_IU reported by Sarah Williams - may be could be closed , it looks like it is because of pool restart problems reported by Charles
  2. Requests from ATALS (Doug Benjamin):
    1. to allow multiple partitions for xrootd cache to be added via vdt configure_xrootd script
    2. add utilized ports list to documentation - done
