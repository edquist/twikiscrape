Second Agenda Item

Release of operation toolkit

Discussion

Abhishek reported that release of new version is scheduled for mid February. It will include some bug fixes and some new tools. One of tools collects and reports via email current dcache status & statistic information. This tool was used successfully at UCSD at some point.

Third agenda item

Gratia probes release

Discussion

Bugs have been found and fixed in the latest release of dcache transfer probe. Chris has built source rpms. Tanya will build a new version of rpms, and published it on VDT dcache web page, so sys admin could just upgrade these rpms on thier dcache installation.

It looks like gratia gridftp probe is ready to be re-tested (more problems was found this week). If the tests succeed Chris will build a new rpm and ask Alain to include it in vdt.

Fourth agenda item

Horst Severini report on BeStman?-gateway installation/tests

Discussion

Horst has installed BeStMan-gateway on top of Ibrix and in general is pleased with performance and functionality. There are several issues with configuration:

  • localPathListAllowed option doesn't work - still allows to write in different area. Alex has confirmed that this is a bug and will try to fix it in next release
  • It will be helpfull if "localPathAllowed" option is added to vdt configure-bestman script for BeStman?-gateway. Horst thinks that would be good in terms of security, too, to only allow srm access to an area you explicitly allow, and forbid everything else.
  • lbnl-srm-client should be able to read the following parameters from user's environment: GLOBUS_TCP_PORT_RANGE=

GLOBUS_TCP_SOURCE_RANGE=

X509_CERT_DIR=

GSI_DAEMON_TRUSTED_CA_DIR=

* Horst also asked us to pass the following request to VDT: "I was requesting that these two VDT packages, wn-client and vdt-client, should be looked at to see if we can improve the configuration of these above env vars a little better during (pacman) installation, since currently they don't seem to be configured at all, which I believe causes problems with the srm tools to work behind firewalls and on compute nodes which don't have /etc/grid-security/certificates accessible -- which I believe is the case in most clusters"

Fourth agenda item

dcache 1.9.2 status

Discussion

dcache 1.9.2-1 has been released. Tanya has built a new vdt package and installed a new version on one node. She has started some preliminary testing. It looks like new pnfs rpm should be also included in this new release (dcache team has not released this rpm yet). There are numerous code changes/new features in this release so a lot of testing will be required before we can release it to vdt.
