First agenda item

Ted's IRODs presentation

Discussion

A powerpoint presentation for use at today's meeting has been added to the docdb page for the preliminary report.

https://osg-docdb.opensciencegrid.org:440/cgi-bin/ShowDocument?docid=967

Below are some questions that have been asked and answered during the meeting:

  • Q. Can you give thumb up or down on your evaluation of iRODS and its usability for OSG?
    A. No definite opinion. This is a question we should answer after we have a thorough discussion
  • Q. How responsive are iRODS support group/developers?
    A. Mailing list is active and looks like problems have been fix fast. They have responded to our questions.
  • Q. Do you envision iRODS running as a central service.
    A. Yes for now, but nothing could preclude us having iRODS install as an appliance on a site used explicitly for public storage. iRODS servers can talk to each other.
  • Q. Should microservices be written in C?
    A. Not necessarily, we can use a wrapper on top of any executable.
  • Q. There is a notion of collection (same path) but not a notion of data set. It could be useful to select all the files in the same data set and file could belong to multiple data sets.
    A. It could be possible to use file metadata to indicate datasets it belongs to.
  • Q. How file transfer via srm could be synced with metadata update?
    A. File transfer via srm should be executed with the custom written script. It will take care of synchronization of metadata.
  • Q. Can we move data without waiting for a user to initiate the request?
    A. No at the moment, we will need to figure out how to manage delegation.

We went through teh half of the slides. We will need to have another meeting when Ted returns from vacation. We have to concentrate on features outlined in blueprint in order to see if iRODS can satisfy all of them.
