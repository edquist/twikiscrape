Fourth agenda item

xrootd/xrootdfs release in vdt status (Tanya)

Discussion

New version of xrootd, xrootdfs, xrootd-gridftp has been installed. Some of the certification tests have failed. The investigation has shown that CNSd server on one of the data node was apparently stuck and failed to report the creation of the new files to the redirector node's composite name space. From the user point of view it looks like a file has been transferred successfully (srm-copy succeeded) but the attempt to list/retrieve this file failed (srm-ls,.. failed). The attempt to restart all the related daemons on the "bad" data server node failed due to "Damaged or invalid /usr/local/xrootd-test/xrootd/var/admin/XrdCnsd.Recovery" file. After consulting with developers the content of admin directory has been deleted and the daemons were successfully restarted. The "missing" files were not visible for user but according to the developers this was an expected behavior. After restart all the certification tests passed. The developers have proposed to try to redo the transfer of "missing" files. The srm-copy succeeded, the content of the file has changed on the data server but the file still remain invisible for a user. An attempt to do the same transfer using root protocol succeeded and the file has appeared at the composite name space.

Developers are looking into this problem.

Fifth agenda item

Abhishek's plans for hadoop testing

Discussion

Goal is to integrate Hadoop-SRM SE technology in OSG Storage and in VDT stack:

  • Build a technology pipeline to evaluate and integrate Hadoop-SRM SE packages into VDT. Improve documentation.
  • Collaborate with Scalability/Reliability/Usability Area to periodically certify performance.
  • Plan and issue 'golden releases' of Hadoop-SRM SE, on 4-6 months timescale, officially supported by OSG Storage.
  • Sustain community support through excellent expertise in osg-hadoop list.
  • As viable, leverage on work of commercial startups already supporting Hadoop FS.
