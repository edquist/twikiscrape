First agenda item

HDFS 0.2 and related software

Discussion

hdfs 0.2 is ready for ITB testing. The release also include BeStMan 2.0.10 and grdiftp-hdfs 0.3.1-1 (with fixes for directory listing). All rpms have been installed on gridwork cluster and tested. Jeff will not be able to work on documentation during the next week but he will provided installation instructions for ITB testing.

Second agenda Item

BeStMan2 2.0.10

Discussion

We are still investigating SocketException? errors that have been observed during testing of BeStMan 2.0.10 client/server. Tanya will send Alex her script so the problem could be reproduced at LBNL. At Fermilab, the client and server are running on VMs. BeStMan2- gateway has the default configuration and uses GUMS for authorization (tested with SAML and XACML). It looks like an exception happens during client communication with jetty. It looks like srm-lbnl client (1.3.18) gets the same exceptions but was able to recover after ~ 10 minutes.

Third agenda item

xrootd native packaging

Discussion

We have released xrootd-server, client, xrootdfs rpms. Doug has finished packaging gridftp-xrootd-dsi. Tanya was able to install and test it. The only problem is that globus-url-copy stops working when configuration is modified in such a way that the VMP has a different name from the xrootd storage path. Doug said that he thinks that the order should be different: redirector:port/VMP=/storage_path. Tanya will try that and if it is true we will need to rebuild rpm changing configuration template file accordingly.
