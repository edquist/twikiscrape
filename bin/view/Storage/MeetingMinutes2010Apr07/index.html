Abhishek: 

* A new update to VDT Hadoop has been made with important bugfixes ( HADOOP-6498, HDFS-596). 
This is currently in ITB YUM repository, and under evaluation by a few sites. Interested sites can switch the choice of repository 
(Production versus ITB) as listed in section 3 of VDT install guide.
* Release Notes have now been added to http://vdt.cs.wisc.edu/components/hadoop.html
* We plan to identify a 'version number scheme' for VDT Hadoop releases. YUM updates can be transparent and are difficult to rollback 
(due to RPMs and standard system locations) so Brian, Mike T, Abhishek are thinking more about how to approach this in long term. 
This is one feature which is significantly different from Pacman.

First agenda item

Documentation

Discussion

Ted is continuing to work on documentation. He is writing https://twiki.grid.iu.edu/bin/view/Documentation/StorageInfrastructureSoftware. He is planning to incorporate Tanya's comments on Site Admin doc as soon as he finishes with Infrastructure software. He has marked all the obsolete documents in twiki. The work is going slowly and could not be our highest priority. It is possible that we will not be able to finish before fall.

Second Agenda Item

Hadoop documentation

Discussion

Doug has installed bestman/gridftp-hdfs/hadoop on two test nodes following the documentation on twiki. He has encountered several problems and sent Abhishek a list with possible improvements. We have discussed some of them during the meeting and Abhishek has agreed to work with Doug on improvements to the document.

Third agenda item

dCache transfer probe

Discussion

Andrew has finished with coding and unit test. Chris has built it and Neha packaged and installed it on gridwork. Unfortunately, the billingAggregator failed with fatal error. Andrew will start testing on gridwork and make sure that the probe is working and collecting right information before asking for new rpms creation. He thinks that he will be able to warp it up within a week. He is also waiting from Brian and Phillipe to respond to his email about implemented features. Brian has promised to respond soon.

Fourth agenda item

BeStMan status

Discussion

Alex is working on BeStMan 2 with jetty. He is also planning to release srm-cleint that will return stack trace in case of exception, this should greatly help with debugging problems.

Fifth agenda item

transfer probes and OSG sites

Discussion

Brian and Karthik are working on gratia transfer daily report. It should go in production on Monday. In the future he is planning to contacting sites about installing the probes in a methodical way and provide a daily view/alert of sites that stop reporting.

Six agenda item

chimera

Discussion

Ted has installed Chimera following dcache instructions. Has encountered several problems, some of the validation tests are still failing. He will redo it again and notify dcache team about the issues he has encountered.
