Install Instructions

  1. Install rsv-vo-gwms package 
    yum install rsv-vo-gwms --enablerepo=osg
  2. Set up a proxy file readable by rsv user and and with vo VOMS extension. It needs to be renewed outside of rsv (possible via a cron job). An example running in the rsv crontab would look like this:
    * */12 * * * /var/rsv/.globus/refresh_proxy.sh </dev/null >/dev/null 2> /dev/null
    And the refresh_proxy script like:
    #!/bin/bash                                                                                                                                                   
voms-proxy-init -cert /etc/grid-security/rsv/rsvcert.pem -key /etc/grid-security/rsv/rsvkey.pem -voms osg -out /etc/grid-security/rsv/mis.proxy -hours 72
  3. Modify file 
    /etc/osg/config.d/30-rsv.ini
    and set 
    enable_gratia = False
    and the rsvcert and rsvkey paths 
    service_cert
    , 
    service_key
    , and 
    user_proxy = etc/grid-security/rsv/mis.proxy
  4. Run 
     osg-configure -v
    and then 
     osg-configure -c
  5. Modify the file 
    /etc/rsv/metrics/org.osg.local-gfactory-querying-local.conf
    to adjust the constraints for the entries that you are interested in testing
  6. Add file 
    /etc/condor-cron/config.d/98_public_interface.config
    with the following contents: 
    NETWORK_INTERFACE = REPLACE_ME_WITH_HOST_IP
  7. Start 
    httpd, condor-cron, rsv
    services.
  8. Run 
     rsv-control --enable org.osg.local-gfactory-querying-local  --host fermicloud053.fnal.gov
    where host can be localhost or the actual name of the node where rsv-vo-gwms is installed.
  9. Run 
    rsv-control --on org.osg.local-gfactory-querying-local  --host fermicloud053.fnal.gov
    where host can be localhost or the actual name of the node where rsv-vo-gwms is installed
