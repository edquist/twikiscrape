Running BLAST

We will try to built on exercises you have been doing for the last two days. You have already submitted a simple script to the GRID that ran BLAST see exercise. In this case BLAST application and databases were already pre-deployed on osg-edu cluster. Let's consider the following problem: you still would like to run your job on osg-edu cluster and you have application pre-deployed on the worker nodes of this cluster, unfortunately the data are not accessible from the worker nodes. The only place from where you can get the data is a SE at OSG-EDU site. You you have to figure out how to solve this problem. Can you do it on your own?

Solution

First, copy a sequence query you will need to use as input to blast executable (copy = yeast.query1= from ==/opt/workshop/dm_exercises/part2==to your working directory) Then, write a simple script that will do the following:
  1. Setup GRID environment
  2. Create a temporary space on the worker node under $OSG_WN_TMP, move to this directory
  3. Find all the files related to the database you want to search (e.g yeast) at SE osgedu (surl: srm://se.sp.grid.unesp.br:8443/srm/v2/server?SFN=/store/osgedu) . The directory path to all BLAST databases is /store/osgedu/blast/db-1/. (Try to do srm-ls command and verify that they are actually there).
  4. Download all the files associated with yeast to the worker node
  5. Run BLAST executable with required parameters (database name, sequence query and log file)
The Condor-G submission file should describe input and output files you want to transfer with these two scripts.

You can try to solve this problem on your own. If you need help you can check my examples. Copy blast.sh script from /opt/workshop/dm_exercises/part2/blast.sh

#!/bin/bash
################################################################################
#  blast_wrapper - download specific blast databases from a particular Storage Element  #
#          execute blast query for a particular query                           #
#          do cleanup                                                           #
#################################################################################
usage ()
{
     log "Usage: ./blast_wrapper.sh surl db_name query_file logfile"
}
log ()
{
if [ x$logname == x ]
then
        echo $1
else
        echo $1 >>$logfile
fi
}
# test if we have four arguments on the command line
if [ $# -lt  3 ]
then
    usage
    exit 1
fi
surl=$1
dbname=$2
query=$3
logname=""
cwd=`pwd`
if [ $# -eq 4 ]
then
        logname=$4
        logfile=$cwd/$logname
        #just in case create a logfile
        touch $logfile
fi
retVal=0
log "Start blast_wrapper.sh"
#setup grid enviroment
. $OSG_GRID/setup.sh
#create temporary directory
export TMPDIR=$OSG_WN_TMP
tmpDir=`mktemp -d -t osgedu.XXXXXX`
cd $tmpDir
log "Current directory is $tmpDir"
#move and change the permission of the uploaded script
mv $cwd/blast.sh .
chmod 755 blast.sh
mv $cwd/$query .
mkdir db-1
#get all the databases we need to download
log "Executing: srm-ls $surl|grep  SRM-CLIENT|grep yeast"
srm-ls $surl|grep SRM-CLIENT|grep $dbname|awk -F'db-1/' '{print $2}'|while read name
do
        log "Executing: srm-copy $surl/$name file:///$PWD/db-1/$name "
        srm-copy $surl/$name file:///$PWD/db-1/$name
        curRetVal=$?
        log "Return value $curRetVal"
        if [ $curRetVal -ne 0 ]
        then
                retVal=$curRetVal
                log "New Return value $curRetVal"
        fi
done
log "Return value $retVal"
if [ $retVal -eq 0 ]
then
        #move word_counter script
        option=""
        if [ x$logname != x ]
        then
                option=$logfile
        fi
        log "Executing: ./blast.sh $dbname $query $option"
        ./blast.sh $dbname $query  $option
        retVal=$?
        log "Return value $retVal"
fi
cd $OSG_WN_TMP
rm -rf $tmpDir
log "End blast_wrapper.sh"
exit $retVal

Storage Discovery

You should know by now enough about storage to be able to try to figure out how to use it on your own. The problem is that we didn't give any tools to figure out what Storage Element will support your VO. Also, how you would find the surl of a SE. You can always use condor_status and various constraints to try to figure out this information, but it is not a trivial task. The OSG has a BDII service which aggregates LDAP information provided by sites. The information reported by each site describes static and dynamic resources available at that site and VOs that are authorized to use these resources. The OSG Discovery Tools allow users to discover what SE supports thier VO and how to access it.

  1. Setup GRID environment
  2. See all the get_ commands available for information discovery:

LCG-UTILS and File Catalog

Let's try to use LCG SRM Client as an ordinary SRM client to move data to and from storage without access to LFC catalog:

  1. Save the blast_results.12.0 into osgedu SE: 
    lcg-cp -b -D srmv2 file://home/user/osg_school/dm_part_2/blast_results.12.0  srm://se.sp.grid.unesp.br:8443/srm/v2/server?SFN=/store/osgedu/user/blast_results.12.0
  2. Verify that results are there: 
    lcg-ls -l  -b -D srmv2 srm://se.sp.grid.unesp.br:8443/srm/v2/server?SFN=/store/osgedu/user/blast_results.12.0
  3. Delete this file: 
    lcg-del -b -l -D srmv2  srm://se.sp.grid.unesp.br:8443/srm/v2/server?SFN=/store/osgedu/user/blast_results.12.0

Now let's try to use LCG Utils commands that allow to access LFC catalog.

  1. Let's first defined the location of LFC catalog for your osgedu:
    export LFC_HOST=uct2-dss.uchicago.edu
  2. Now, let's create a subdirectory in LFC catalog:
     lfc-mkdir -p /grid/osgedu/user
  3. Then upload the file to osgedu SE and register it in the catalog with the name "blast_results":
    lcg-cr    -v  --vo osgedu  \
    -b -T srmv2 \
    -l lfn:/grid/osgedu/user/blast_results\
    -d srm://se.sp.grid.unesp.br:8443/srm/v2/server?SFN=/store/osgedu/user/blast_results.12.0 \
    file:////home/user/osg_school/dm_part_2/blast_results.12.0
  4. Verify that the file is registered in the catalog
    lfc-ls /grid/osgedu/user/blast_results
  5. Delete file from the SE and remove it from the LFC catalog 
    lcg-del -b -D srmv2  srm://se.sp.grid.unesp.br:8443/srm/v2/server?SFN=/store/osgedu/user/blast_results.12.0
