About this document

SRM-client toolkit is a full implementation of SRM V2.2, developed by Lawrence Berkeley National Laboratory as generic SRM v2.2 clients.

This document is intended for users who want to use LBNL SRM clients to do any of following

  • list files at a storage location
  • get files from a storage location
  • put files in a storage location (in usual or space reserved area)
  • transfer files from one storage location to another
  • create/remove directory at a storage location
  • make/update/release space reservation

Introduction

SRM-client toolkit is a full implementation of SRM V2.2, developed by Lawrence Berkeley National Laboratory as generic SRM v2.2 clients. They have been tested for all current SRM v2.2 implementations such as BeStMan, CASTOR, dCache, DPM, SRM/SRB and StoRM. These commands get tested continuously for compatibility and interoperability.

Engineering Consideration

These clients do not work with SRM v1.1 server

Requirements

  • A valid grid proxy or voms proxy
  • Permission to actually get/put/list the data (You should be allowed to actually retrieve/store the data on the storage element.)
  • OSG Client, CE or WN software installed.
    • Note: the actual RPM needed is bestman2-client (with associated dependencies), which you can install with "yum install bestman2-client". However, installing one of the above supported software packages is recommended.
  • The url for the Storage Element you are trying to access

Procedure

LBNL client toolkit is packaged in to VDT and should be available on any node that has the OSG Client, CE or WN software installed. To run the clients, you need to
  • login to proper node
  • generate your voms credentials
  • bestman2-client RPMs install the clients in the system path (/usr/bin), but you can verify they are in your path by running: 
which srm-copy
Notes
  • Command line options available for each command can be listed by using -help
  • The url for Bestman based SE is of form - srm://[HOSTNAME]:[PORT]/srm/v2/server
  • The url for dCache based SE is of form srm://[HOSTNAME]:[PORT]/srm/managerv2
  • The output for the same command run against different type of SE may vary

srm-mkdir - Create a new directory on remote storage element

Syntax 
srm-mkdir sourceurl [command line options]
srm-mkdir [command line options] -s sourceurl
srm-mkdir [command line options] -s sourceurl -s sourceurl
srm-mkdir [command line options] -f file
where file is the path to the xml file containing the sourceurl information. For requests with more than one file -f option is used.

Example

Create a new directory More... Close 

-bash-3.00$ srm-mkdir srm://fndca1.fnal.gov:8443/srm/managerv2?SFN=/pnfs/fnal.gov/usr/fermigrid/volatile/fermilab/ns2010

Syntax 
srm-copy sourceUrl targetUrl [command line options]
srm-copy [command line options] -f file
