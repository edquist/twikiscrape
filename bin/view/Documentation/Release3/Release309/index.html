OSG Software release 3.0.9

Release Date: 27-Mar-2012

Summary of Changes

  • glideinwms: version updated to 2.5.6.
  • Multiple Globus bugfixes, including:
    • SOFTWARE-561: Kill running globus-job-manager processes on upgrade to avoid problems with different versions of the job-manager running concurrently.
    • GRAM-319: Fix job managers not being able to shut down when a proxy expires.
    • Fix for recurring segfault for a Globus user if Condor creates an empty log file for any of the user's jobs.
    • GRAM-309: Fix for gatekeeper not starting up on machines that do not have IPv6 enabled.
    • GRAM-315: Fix to prevent job loss on globus-job-manager restart for gliteWMS jobs.
    • GRAM-317: Fixes random failures if the user proxy is larger than 16KB.
  • gratia-probe: The correct dependencies for the Gratia probes are now installed on the Compute Element, and a bug has been fixed in the Gratia GridFTP? probe.
  • osg-cleanup: SOFTWARE-561 more accurately delete old Globus files; avoid cleaning up per-job-manager lock files.
  • osg-configure can now configure Gratia probes on the CE, does a better job of configuring lcmaps for authorization, and has minor bug fixes.
  • xrootd: updated to 3.1.1

All of the documentation can be found in the "Release3" area of the Twiki.

Updating to the new release

If you have an existing installation, you can update to this new release with: 

[root@client ~]$ yum update

NOTE
Please be aware that running yum update may also update other RPMs. You can exclude packages from being updated using the --exclude=[package-name or regex] option for the yum command.
