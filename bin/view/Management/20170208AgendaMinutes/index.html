Minutes

Attendees:

Status Updates

  • Fsurf, an implementation of Freesurfer for OSG, is basically ready to ship (pending approval by Don Krieger).
  • OSG has helped the Xenon1T project a great deal get set up for production work, and they are now taking data.
  • South Pole Telescope (SPT) also just started taking data, and also involved a fair amount of set up work. Now, the main follow-up work is automating data transfer to NERSC, and maybe helping them with their analysis workflow, which is derived from a workflow that Benedikt worked on for IceCube. There was a question about how Globus Online was selected to be their main data transfer system, and mostly it came down to being the path of least resistance.
  • There are several sites queued up for Hosted CEs, and the User Support team just wants to get some good documentation in place first.
  • The team has a couple training events lined up this spring, and possibly a couple more for the summer.

Concerns

  • There just are not as many opportunistic resources available these days, and some user groups are noticing. In some cases, there may be fewer total resources available, and also there are more groups using more cycles consistently. We (OSG) have the ability to give a group special priority, but doing so is fairly rare and is decided at the Executive Director level. Brian B. noted that we may have the technical ability to implement fair share at the group (e.g., LIGO), rather than individual, level in the near future, but that mechanism is not available yet today. In the end, simply having more total resources will always improve the situation.
