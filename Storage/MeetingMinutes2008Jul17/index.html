Agenda item


1. Status/features of BeStMan upcoming major release in August (Alex)

Upcoming major release will be in VDT in August. It will allow to install and configure BeStMan with xrootd. It will still possible to configure just BeStMan (default configuration). If BeStMan and xrootd are installed - it will impossible to request storage reservation via srm. Opportunistic storage may be allocated by admin (changes in BeStMan configuration files) and then users could use a space token provided by admin to use an allocated space.

2. The progress with storage GIP for BeStMan (Alex & Brian)

5. Ruth has asked to discuss "Storage accounting repositories". I hope Brian and/or Ted have some information
what it is all about.
Ted & Brian thought that Ruth was asking about gratia storage probes. If so, they are part of dcache distribution and should be enable on dcache cluster by dcache admin. The probes are reporting to gratia central server. There is no gratia storage "ready" reports, one should do sql query to get information. Brian was planning to write them but this task has very low priority on his list.
