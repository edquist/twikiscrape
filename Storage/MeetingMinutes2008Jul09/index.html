Storage Meeting Minutes 2008 Jul 09

Attending

  • Andrew Baranovski
  • Brian Bockelman
  • Ted Hesselroth
  • Neha Sharma
  • Alex Sim
  • Rob Quick
  • Tanya Levshina

Agenda items

1. The Dynamic Megatable and Space Usage monitoring project.
Brian provided brief overview of the project that is started by several groups within WLCG. The goal of the project is to puto publish information for management and VO administrators with a view of the installed capacity and resource usage by the VOs at sites, storage partitioning etc. Brian is planning to attend meetings and will report the requirements (what storage information should be provided by various storage systems, e.g dcache, BeStMan,..) and the usability of the developed tools for OSG. We have discussed the relevance of Andrew's dashboard prototype to this project. It looks like that his approach could be used to achieve this project goals.

2. Storage GIP for BeStMan
Alex and Brian have started working on including into GIP missing BeStMan information. Tanya suggested that this should have high priority. There are numerous sites (10?) that are using BeStMan. They will not able to advertise their storage correctly. The fix should be included into vdt release and we will ask these sites to do an upgrade.

3. Issues (if any) with D0 utilization of OSG storage
Andrew did presentation about D0 usage patterns of OSG opportunistic storage for CMS. UCSD (CMS) has promised to allocate some storage for D0.
D0 processing issue: time discrepancy between d0 node and some dcache pool node caused job failures.

4. GIP validation/client for storage advertising
Have discussed the necessity of gip validation. Tanya promised to send the problems she has found to Brian.
Tanya is starting to work on the primitive client to display storage information in human readable format. It is
not a real solution and we have to see if the Megatable project solves this problem.

5. Tickets
Discussed with Rob if we have to open GOC ticket when the problem has been reported by a user and we have opened the ticket in dcache for that problem. Rob said that all problems reported by users should be in GOC.
