Campus Grids - Rob Gardner

Key initiatives

  • Distributed Environment Modules
    • 57 packages now installed. 61 applications from the Xsede campus bridging list, of which 26 are targeted for DHTC on OSG. We have 9 of those currently in OASIS.
    • Instructions and current inventory, documentation: https://confluence.grid.iu.edu/display/CON/Distributed+Environment+Modules
  • More development on "tutorial" command
    • tutorial modules maintained in github and connect-book
    • Available on login.osgconnect.net:
      $ tutorial
usage: tutorial name-of-tutorial
       tutorial info name-of-tutorial

Available tutorials:
htcondor_transfer  Files for tutorial on using HTCondor file transfers
mathematica        Basic Mandelbrot set calculation using Mathematica
namd               Tutorial - NAMD (NAnoscale Molecular Dynamics)
oasis_parrot       Files for tutorial on using OASIS through Parrot
octave             Matrix diagonalization using GNU Octave
pegasus            Tutorial introducting the Pegasus workflow management system
photodemo          Storage access demonstration using distributed photograph analysis
quickstart         Basic HTCondor job submission tutorial
R                  Application Example - R (statistical analysis)
scaling            Job steering and scaling tutorial
software1          Software access using HTCondor or the Web
stash_chirp        Files for tutorial on accessing stash using Chirp
stash_http         Files for tutorial on accessing stash over HTTP
swift              Swift parallel scripting for OSG Connect Users
userschool2014     OSG Connect exercise for OSG User School 2014
  • Connect client development
    • Goal is to give campus grids a client tool:
      • Pre-installed, configured, and supported by local campus cluster admin
      • Submission to OSG via grid universe
      • Submission to local scheduler via vanilla universe, using local campus allocation
      • "connect" and "tutorial" commands
      • modules command: allow switching between local modules collection and OASIS modules
  • Software Carpentry + OSG Campus Bootcamp
    • Working with User Support and Lauren Michael to hold co-branded training event January 15-16, 2015 (Madison)
    • Targeted for new users to scientific computing: go from an intro to the Unix command line to running jobs on OSG in 2.5 days.
    • Preliminary syllabus: http://dmbala.github.io/2014-09-29-UC-Test/
  • Working with Syracuse (Orange Grid), expanded discussion to include local LIGO group. Signed them up to evaluate OSG Connect.

Top Issues/Concerns

  • Preparing for SWC-OSG bootcamp
  • Organizing tutorial modules for re-use, e.g. ACI-REF webinars

Key Accomplishments
