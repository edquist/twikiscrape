Usability Test Security: 'How to Obtain a Grid Certificate for a new User'

  • https://twiki-docteam.grid.iu.edu/bin/view/ReleaseDocumentation/CertificateUserGet
    • this is too long, imagine a student or scientist who wants to get to a certificate quickly.
    • some VOs handle certificate requests differently (LIGO,Engage) please list these exceptions
    • What if I have Windows computer (students!)?
    • document format violations
    • how do I import the certificate into my browser?
    • explain that the proxy expires and needs renewal
  • https://twiki-docteam.grid.iu.edu/bin/view/ReleaseDocumentation/CertificateGetCmd
    • people who land on this page do not have a VDT installation and won't succeed
    • I don't know yet, if I'll use the command line or a web interface to the grid
    • host and service certificates are documented better elsewhere
  • https://twiki-docteam.grid.iu.edu/bin/view/Documentation/CertificateGetWeb
    • why do I have to close all tabs?
    • can I use another browser than the ones listed?
    • highlight the requirements, so they stick out of text
    • make a real checklist, that people can click off ( well, not sure that is possible in Twiki )
    • what is a DN anyway?
    • this step is missing: In the top navigation bar, click Retrieval. - In the left navigation bar, click Import CA Certificate Chain.
    • why does engage have almost identical information: https://twiki.grid.iu.edu/bin/view/Trash/Trash/Engagement/EngageNewUserGuide
    • the command line presentation is wrong, see document formatting standards
    • how do I check that my certificate is valid?
    • ssl commands for Unix command line shown, but what about Windows and Mac OSX?
    • plugins that run the openssl command in the browser for you?
    • what is next? Where do I copy this too? What is the submit host?
    • add References at the end of document to direct the reader to what to do next
  • https://twiki-docteam.grid.iu.edu/bin/view/ReleaseDocumentation/UserCertificate
    • Who needs this and why?
    • The concept of a VOMS server and certificate upload is not explained at all, readers will be lost with this document.
