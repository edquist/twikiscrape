OSG Software Release 3.1.23

Release Date: 2013-09-10

Summary of changes

Changes include:

  • Java 7 migration
    • Please refer to the documentation in Installing OSG Software to Use OpenJDK 7
  • VO package v47
  • Initial release of osg-control
  • Handle CAs and CRLs in OASIS
  • RSV support for HTCondor-CE

Also in the upcoming repository:

  • HTCondor-CE tested with HTCondor (not tested or supported with PBS)
  • HTCondor 8.0.2

Detailed changes are below. All of the documentation can be found in the Release3 area of the TWiki.

12 JIRA tickets were addressed in this release.

Known issues

osg-configure: Variables defined by user in one .ini file do not carry over to subsequent .ini files
osg-control: Due to an existing bug in osg-configure, you won't be able to restart tomcat6 on EL6 systems through osg-control.
Java 7: IBM's JDK (java-1.7.0-ibm) is known to not work with BestMan 2.

Updating to the new release

You can update to this new release with: 

[root@client ~]$ yum update

NOTE
Please be aware that running yum update may also update other RPMs. You can exclude packages from being updated using the --exclude=[package-name or glob] option for the yum command.

NOTE
Watch the yum update carefully for any messages about a .rpmnew file being created. That means that a configuration file had been editted, and a new default version was to be installed. In that case, RPM does not overwrite the editted configuration file but instead installs the new version with a .rpmnew extension. You will need to merge any edits that have made into the .rpmnew file and then move the merged version into place (that is, without the .rpmnew extension). Watch especially for /etc/lcmaps.db, which every site is expected to edit.

Need help?

Do you need help with this release? Contact us for help.

Detailed changes in this release

The OASIS install of the tarball client now uses the CVMFS variable OASIS_CERTIFICATES to choose the directory containing CA certificates and CRLs. The value for this directory is set in your CVMFS configuration files. Installing the latest version of oasis-config, included in this release, will provide a default value for OASIS_CERTIFICATES that points to a certificate directory on OASIS.
