About this Document

This document is for SRM Storage end users. It provides some usage examples of LCG Data Management Client Tools.

This document applies to the latest release OSG-3.x

on on

Introduction

LCG Utils is a suite of client tools for data movement written for the LHC Computing Grid. The tools are based on the Grid File Access Library, which is also included. The tools allow users to copy files between CE (Compute Element), WN (Worker Node) and a SE (Storage Element) and to register entries in the file catalog and replicate files between SEs. Some commands use logical file names and require a connection to a BDII-based catalog; exceptions are file copies and deletions, which take endpoints based on the SRM URL. This document does not contain examples of commands that require access to file catalog. The usage of such command is explained in the LCG User Guide.

This document is intended for users who want to use lcg-utils clients to do any of following

  • list files at a storage location
  • get files from a storage location
  • put files in a storage location (in volatile or space reserved area)
  • transfer files from one storage location to another .

How to get Help?

If you cannot resolve the problem, the best way to get help is by following User Help. If you cannot find answers, please send all your questions to :
  • osg-storage@opensciencegrid.org

Requirements

  • A valid grid proxy or voms proxy
  • Permission to actually get/put/list the data (You should be allowed to actually retrieve/store the data on the storage element.)
  • OSG Client, CE or WN software installed.
    • Note: the actual RPM needed is lcg-utils (with associated dependencies), which you can install with "yum install lcg-utils". However, installing one of the above supported software packages is recommended.
  • The url for the Storage Element you are trying to access
  • Some of the program may require access to a BDII containing information about the SE

Using LCG Utils commands

LCG Utils toolkit is packaged in to OSG and should be available in the default PATH on any node that has the OSG Client, CE or WN software installed.

You can verify they are in your PATH by running: 

[user@client ~]$ which lcg-cp
Notes
  • Command line options available for each command can be listed by using --help
  • The url for Berkeley Storage Manager (BeStMan) based SE is of form - srm://[HOSTNAME]:[PORT]/srm/v2/server
  • The url for dCache based SE is of form srm://[HOSTNAME]:[PORT]/srm/managerv2
  • The output for the same command run against different type of SE may vary

After setting environment variables and generating voms proxy certificate, you can try to execute basic commands.

lcg-cp command

This command is used to perform file transfers.

lcg-cp will create any missing directories if needed.

Options can be listed as follows 

[user@client ~]$ lcg-cp --help
Usage:  lcg-cp [-h,--help] [-c,--config config_file] [-n nbstreams] [-b,--nobdii] [-D,--defaultsetype se|srmv1|srmv2]
        [-T,--srcsetype se|srmv1|srmv2] [-U,--dstsetype se|srmv1|srmv2] [-s,--sst src_spacetokendesc]
        [-S,--dst dest_spacetokendesc] [--connect-timeout value] [--sendreceive-timeout value]
        [--bdii-timeout value] [--srm-timeout value] [--checksum] [--checksum-type type]
        [-v,--verbose] [-V,--vo vo] [--version] src_file  dest_file

Copying a local file to the remote SRM endpoint

Syntax 

[user@client ~]$ lcg-cp -v -b -D srmv2 file://local_file SURL
where

local_file - absolute location of source file beginning with a "/"

SURL - remote storage file location

Example 

[user@client ~]$ lcg-cp -v -b -D srmv2 file:///tmp/test   srm://fgitb315.fnal.gov:10443/srm/v2/server?SFN=/cache/test/test100

Copying file in to a reserved space area

In order to copy file to a reserved space area you need to know a space token description.

Syntax 

[user@client ~]$ lcg-cp -v -b -D srmv2 file://local_file SURL -s space_token_description

where

local_file - absolute location of source file beginning with a "/"

SURL - remote storage file location

Example

Let's assume that space token with description MY_SPACE has been already reserved for you on SRM endpoint you are using. 

[user@client ~]$ lcg-cp -v -b -D srmv2 file:///tmp/test   srm://gw015k1.fnal.gov:8443/srm/v2/server?SFN=/data/xrootdfs/public/test/test100 -s MY_SPACE

lcg-del command

This command is used to delete a file.

LCG Utils toolkit doesn't provide a means to delete a directory.

Options can be listed as follows 

[user@client ~]$ lcg-del --help
Usage: lcg-del [-a] [-c,--config config_file] [-h,--help] [--force] [-b,--nobdii] [-d,--dir] [--srm-timeout value]
        [--connect-timeout value] [--sendreceive-timeout value] [--bdii-timeout value]
        [-D,--defaultsetype se|srmv1|srmv2] [-T,--setype se|srmv1|srmv2] [-l,--nolfc]
        [-s,--se se] [-v,--verbose] [-V,--vo vo] [--version] (-f file | surl [...])

Syntax 

[user@client ~]$ lcg-del -b -v -l  -D srmv2 SURL

where

SURL - remote storage location of file you want to delete

Example 

[user@client ~]$ lcg-del -b -v -l -D srmv2  srm://gw015k1.fnal.gov:8443/srm/v2/server?SFN=/data/xrootdfs/public/test/test1/test2/test100
VO name: fermilab
SE type: SRMv2
srm://gw015k1.fnal.gov:8443/srm/v2/server?SFN=/data/xrootdfs/public/test/test1/test2/test100 - DELETED

lcg-ls command

This command is used to list contents of a directory.

Options can be listed as follows 

[user@client ~]$ lcg-ls --help
Usage: lcg-ls [-l] [-d] [-D,--defaultsetype se|srmv1|srmv2] [-T,--setype se|srmv1|srmv2]
        [--connect-timeout value] [--sendreceive-timeout value] [--bdii-timeout value]
        [--srm-timeout value] [-b,--nobdii] [-V,--vo vo] [-h,--help] [--version] file [...]
-l option here specifies a long listing format

Syntax 

[user@client ~]$ lcg-ls -b -v -l  -D srmv2 SURL

where

SURL - location of remote directory whose contents you want to list

Examples 

[user@client ~]$lcg-ls -ld -v -b -D srmv2   srm://fgitb315.fnal.gov:10443/srm/v2/server?SFN=/cache/test/test12
SE type: SRMv2
-rwxr-xr-x   1     2     2      15               ONLINE /cache/test/test12

[user@client ~]$ lcg-ls  -v -b -D srmv2   srm://fgitb315.fnal.gov:10443/srm/v2/server?SFN=/cache/test
SE type: SRMv2
