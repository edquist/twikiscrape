NOTE
The following sections are intended to be a super set for an How To (or Procedure) Document. Particular sections may not apply to all documents, but use all the sections that make sense for the procedure you are documenting.

NOTE
Items in common with many procedures should have their own page and be pointed to by or included in this document.

NOTE
Create a new How To document if some software or operational change requires more than a few changes to the procedure and the old procedure still needs to be available for use.

About This Document

Describe the intended audience or primary user for the document.

Include a brief description of object of this procedure and how this procedure fits into any larger workflow or process.

The applicable environment for use of this procedure. This might include the OSG release for which this procedure applies and the type of site, VO, or User that is is designed for.

Requirements

An in-line numbered list of items assumed to have been already performed before you follow this procedure. E.g.:
  1. Security items (need a Certificate)
  2. Membership in a VO
  3. Gotten access by being a recognized user
  4. Etc.

Procedure

Begin with an overview or list of the high level steps so the reader understands the overall procedure.

This can be followed by a more thorough description of the steps of this procedure (including the links to forms, web pages, commands and options required).
