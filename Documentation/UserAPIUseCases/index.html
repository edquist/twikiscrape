Charge

This group focuses on the providing “API” used for communicating with the networking dashboard.

The initial goal of the group is to understand the most common use cases foreseen for the use of the dashboard and provide API that enables access to the information in dashboard.

In the current scheme the dashboard information is share in the form of JASON objects. This information is accessible through REST interfaces. This group also needs to work closely with the AuthN?/AuthZ group to make the information access secure.

Questions to be addressed by the group

  • Which user communities interested in accessing the networking dashboard?
  • What information is the user communities interested in accessing?
  • What are use cases for accessing the dashboard information?
  • What information is currently missing and not yet provided by the dashboard?
  • What functionalities are missing in the dashboard?

User Communities

Following user communities/projects have been identified as initial group that are interested in accessing information from the networking dashboard.

This group will identify new user communities and work with the communities to identify their needs to make sure that their use cases are satisfied.

Information Accessibility

The networking dashboard information is currently available in the form of JASON objects via REST interfaces. This keeps the design modular and makes it easy to implement a presentation layer that consumes the information and displays it in Graphical format. REST interfaces make it easy to access the information programmatically or through a web browser.

This group needs to understand -

  • If there is a need to make the information available in a format different than JASON objects
  • If there is a need to make information accessible in different format
  • Set of libraries or APIs needed to provide secure access to the information
  • Making the information access scalable
